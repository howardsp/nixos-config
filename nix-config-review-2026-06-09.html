Nix Configuration Review

Repository: ~/nixos-config  |  Date: 2026-06-09  |  Hosts: ruger, avalanche, virtualnix (NixOS 26.05) · nixbookair (nix-darwin 26.05)

Executive Summary

This is a well-organized flake: a clean two-function builder layer (lib/default.nix), clear common/linux/darwin module split, per-host feature toggles, pinned Homebrew taps, and proper follows on home-manager and nix-darwin. Both the ruger and nixbookair configurations evaluate successfully as of this review.

The most important items found: a broken-on-macOS shell startup (direnv hook with no direnv installed), a dead v4l2loopback module on ruger, several root-equivalent privileges granted by default (docker/lxd groups, trusted-users @wheel, PostgreSQL superuser on every host), and mitigations=off on internet-browsing workstations. Documentation fixes (stale igloo host name, wrong file headers, undocumented ports/exceptions) have already been applied to the repo as part of this review.

1 · Flake & Architecture Review (NixOS expert pass)

#SeverityFindingRecommendation
F1High hosts/ruger/default.nix: v4l2loopback is in boot.kernelModules but the matching boot.extraModulePackages line is commented out — the virtual webcam module is never built for kernel 7.0, so the module load silently fails. Re-enable boot.extraModulePackages = [ config.boot.kernelPackages.v4l2loopback ]; (use config.boot.kernelPackages, not a hard-coded kernel version, so it tracks kernel bumps — avalanche hard-codes linux_6_18 and will break on its next kernel change).
F2High lib/default.nix sets nixpkgs.config.allowBroken = true globally for all Linux hosts, silencing breakage for every package. Remove it; if one package is broken, allow it specifically (allowBrokenPredicate or an overlay) and document which one.
F3High system.stateVersion = "25.11" (modules/linux/default.nix) and home.stateVersion = "25.05" (home/common) are defined in shared modules. stateVersion is a per-machine, set-once value; a future host added to this flake would silently inherit an old stateVersion. Move system.stateVersion into each hosts/<name>/default.nix and home.stateVersion per-user/host (or document loudly that new hosts must override it).
F4Medium Duplicate Nix daemon settings: nix.gc and warn-dirty are set in both modules/common/default.nix and modules/common/nix-settings.nix. Consolidate everything into nix-settings.nix; also add nix.gc.dates (Linux) / interval (Darwin) explicitly so GC cadence is documented.
F5Medium No formatter, checks, or devShells flake outputs, despite shipping alejandra/statix/deadnix as packages. statix reports ~25 style warnings (repeated keys, empty lambda patterns); deadnix reports many unused lambda args. Add formatter.<system> = pkgs.alejandra; and a checks output (or treefmt-nix + pre-commit) running alejandra/statix/deadnix, then nix flake check in CI (GitHub Actions).
F6Medium llm-agents and claude-desktop inputs don't follows your nixpkgs — the lock file carries three extra nixpkgs revisions (nixpkgs_2, nixpkgs_3), inflating eval time and downloads. Add inputs.nixpkgs.follows = "nixpkgs-unstable" (or stable) to those inputs where they support it.
F7Medium Home Manager has no backupFileExtension; a switch fails hard whenever a target dotfile already exists. Set home-manager.backupFileExtension = "hm-bak"; in mkHomeManager.
F8Medium features.qemu.enable only gates client packages; virtualisation.libvirtd/virt-manager/SPICE are re-enabled manually in each host, duplicating 4 lines per host and letting toggle and service drift apart (avalanche has libvirtd on with the feature off). In virtualisation.nix: virtualisation.libvirtd.enable = config.features.qemu.enable; etc. (keep lib.mkDefault for overrides).
F9Medium Activation script does ln -sf /bin/sh /bin/bash — an impure global mutation; scripts using bash-isms get sh-mode bash and can break subtly. Link the real bash: ln -sf ${pkgs.bash}/bin/bash /bin/bash, or better use services.envfs.enable = true; which solves all hard-coded shebangs declaratively.
F10Medium Root-run activation scripts edit files in the user's home (Citrix ini, Zoom conf) — wrong layer, can change file ownership, and runs on every system activation. Move to Home Manager home.activation (runs as the user) or generate the config files declaratively with home.file.
F11Low ydotool-based GNOME keybindings (remote paste, close-tab, Citrix keys) require the ydotoold daemon, but services.ydotool.enable is commented out in gnome.nix — those bindings are currently dead. Enable services.ydotool.enable = true; and add the user to the module's group, or remove the bindings.
F12Low xscreensaver service (home/linux) does nothing under GNOME Wayland, which uses its own locker. Remove it, or keep only for X11 sessions.
F13Low EDITOR is set three ways: system EDITOR=vim, HM EDITOR=nvim, and programs.neovim.defaultEditor=true. Eval also prints a programs.neovim.withPython3 deprecation warning. Pick one (suggest defaultEditor = true only), delete the rest, and set withPython3 explicitly to silence the warning.
F14Low zsh history-substring-search is fetched from GitHub by hand; p10k is sourced manually. nixpkgs packages both. Also extensionsFromVscodeMarketplace [] is dead code. Use pkgs.zsh-history-substring-search / programs.zsh.plugins with nixpkgs sources; drop the empty marketplace list.
F15Low home/linux/gnome-settings.nix hard-codes /home/howardsp/... in brightness/paste keybindings while everything else is parameterized by username. Use ${config.home.homeDirectory}.
F16Low 72 Nerd Fonts + ~20 font families install on every host, including the headless-ish VM — slows rebuilds/closure. Split a “fonts-full” vs “fonts-minimal” set, or gate behind a feature toggle.

2 · Linux ⇄ macOS Cohesion

#SeverityFindingRecommendation
M1Critical macOS shell startup is broken by design: home/common/shell/default.nix runs eval "$(direnv hook zsh)" (and bash equivalent) on both platforms, but direnv is only installed via modules/linux/packages.nix and programs.direnv is only enabled in home/linux. On nixbookair every new terminal errors (direnv: command not found) unless Homebrew happens to provide it. Move programs.direnv = { enable = true; nix-direnv.enable = true; }; to home/common and delete the manual eval hooks entirely — the HM module wires shell integration itself on both OSes. Same for the manual fzf --zsh eval: programs.fzf already integrates it.
M2High homebrew.onActivation.autoUpdate = true; upgrade = true; contradicts the pinned, immutable taps (mutableTaps = false, taps as flake inputs). Every rebuild pulls whatever is newest from the cask CDN — your Mac drifts while Linux stays pinned. Set autoUpdate = false and run upgrades deliberately via nix flake update + rebuild, restoring "the lock file describes the machine" on both platforms.
M3Medium The feature-toggle system (features.*) exists only on Linux. The Mac has no equivalent switchboard, so host-level customization diverges structurally. Define the features options in modules/common, and let darwin modules (homebrew casks lists) honor the same toggles (e.g. features.browsers selecting casks).
M4Medium nix.settings.sandbox and store optimisation are commented out globally “because Darwin doesn’t like it”, losing them on Linux too. Guard per-platform: in modules/linux set nix.settings.sandbox = true; and nix.optimise.automatic = true;; leave Darwin defaults alone.
M5Medium Rebuild ergonomics differ per OS (nixlinux vs nixdarwin/nixdarwin-activate aliases). Add one rebuild script in shell-scripts.nix that branches on uname — same muscle memory everywhere (you already ship nh, which also abstracts this: nh os switch / nh darwin switch).
M6Low VS Code is configured declaratively on Linux (HM module) but imperatively on macOS (cask) — settings/extensions drift. Move vscode.nix to home/common (the HM module works on Darwin) or accept divergence consciously and note it in README.
M7Low Linux user account declares shell = pkgs.zsh and groups; the darwin user block sets only home. Fine today, but parity items (shell, description) are implicit on macOS. Optionally set users.users.<name>.shell = pkgs.zsh; on Darwin too for symmetry.

3 · Security Review (Linux security expert pass)

#SeverityFindingRecommendation
S1High mitigations=off on ruger and avalanche disables all Spectre/Meltdown-class CPU mitigations on machines that run four browsers, Zoom, and remote-work tooling — i.e. plenty of untrusted code. Remove it. On Zen 3+/recent kernels the measured gain is typically 1–3% for desktop loads; the README now carries a warning either way.
S2High The user is in docker and lxd groups — both are documented root-equivalents (trivial host-root via bind mounts). LXD isn’t even configured anywhere in this repo; media/plugdev are also unused on NixOS. Drop lxd, media, plugdev. For Docker, prefer rootless mode (virtualisation.docker.rootless.enable = true; setSocketVariable = true;) or switch to podman (already the OCI backend for Home Assistant).
S3High nix.settings.trusted-users = [ "root" "@wheel" ]: any wheel user can feed the daemon arbitrary substituters/unsigned paths — effectively root without a sudo prompt, weakening wheelNeedsPassword. Restrict to [ "root" ] (plus your user only if you genuinely need to add caches ad-hoc; flake-registered substituters keep working for everyone regardless).
S4Medium PostgreSQL runs on every Linux host with the desktop user as SUPERUSER. Local-only listener, but it's un-gated attack surface and privilege. Gate behind features.postgres.enable, grant only CREATEDB/ownership of the paperclip DB instead of superuser.
S5Medium Ollama binds OLLAMA_HOST=0.0.0.0. The Ollama API has no authentication; anyone on the LAN could run/pull models and read responses. (Today port 11434 isn’t opened in the firewall, so the binding's intent is unmet — the config is internally inconsistent.) Decide: local-only → drop the env var (defaults to 127.0.0.1); LAN service → keep it, open 11434 only on trusted interfaces, and front it with a reverse proxy w/ auth if anything sensitive flows through it.
S6Medium Deskflow/Synergy ports 24800–24801 are opened on all hosts via the base networking module, plus per-host re-opens. A software-KVM stream (keystrokes!) reachable from the whole LAN. Open the ports only on hosts that actually share input, verify Deskflow TLS is enabled, and consider networking.firewall.extraCommands to restrict source IPs to the peer machine.
S7Medium Quick-paste keybindings type the contents of ~/.config/.10.txt/.11.txt into whatever window has focus. If these are credentials, they live in plaintext and can be replayed into the wrong window with one mis-chord. Use a secret manager (Bitwarden is already on the Mac; rbw/GNOME keyring + a rofi picker on Linux), or at minimum chmod 600 and accept the risk consciously.
S8Medium Home Assistant container runs with --network=host and a serial device passthrough, on a mutable :stable tag — silent supply-chain updates with host networking. Pin the image by digest (image = "ghcr.io/home-assistant/home-assistant@sha256:...") and update it via PRs; host networking is normal for HA discovery but worth documenting.
S9Medium permittedInsecurePackages = [ "libsoup-2.74.3" ] — EOL library with published CVEs in HTTP parsing. Identify the consumer with nix why-depends (comment added in repo) and re-test removal after every nix flake update.
S10Low OpenSSH posture is good (key-only, no root, no X11) but is enabled on every host including the VM and laptops; no AllowUsers, banner, or fail2ban. Add KbdInteractiveAuthentication = false;, AllowUsers = [ "howardsp" ];, and enable sshd per-host instead of fleet-wide. services.fail2ban.enable = true; if any host is ever exposed.
S11Low Kernel hardening is a good start but partial. Consider: kernel.kptr_restrict=2, kernel.unprivileged_bpf_disabled=1, net.core.bpf_jit_harden=2, kernel.yama.ptrace_scope=1, security.protectKernelImage=true, security.sudo.execWheelOnly=true, and services.fwupd.enable=true for firmware updates.
S12Low Both earlyoom and systemd's default systemd-oomd can be active — two OOM killers with different policies racing under memory pressure. Pick one: keep earlyoom and set systemd.oomd.enable = false;.
S13Low macOS: Touch ID for sudo is enabled (good), but the application firewall isn't managed declaratively. Add networking.applicationFirewall = { enable = true; blockAllIncoming = false; enableStealthMode = true; }; (nix-darwin) to nixbookair.
S14Info Flatpak is enabled with no declared remotes — apps installed there bypass Nix review/pinning entirely. If only Flathub is used, document it; consider nix-flatpak for declarative flatpaks, or drop Flatpak if unused.
